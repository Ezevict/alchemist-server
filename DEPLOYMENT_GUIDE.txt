STEP 1 — GET YOUR API KEYS (5 minutes)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A) Claude API Key
   1. Go to: https://console.anthropic.com
   2. Sign up / Log in
   3. Click "API Keys" → "Create Key"
   4. Copy the key (starts with sk-ant-)

B) Telegram Bot Token
   1. Open Telegram → search @BotFather
   2. Send: /newbot
   3. Choose a name: e.g. "Alchemist Signals"
   4. Choose a username: e.g. "alchemist_signals_bot"
   5. Copy the token BotFather gives you

C) Telegram Chat ID
   1. Start your bot (click the link BotFather gives)
   2. Send any message to it
   3. Open Telegram → search @userinfobot → send /start
   4. It replies with your Chat ID (a number like 123456789)
   NOTE: For a group/channel, add the bot to the group,
         send a message, then use the negative ID e.g. -100123456789

STEP 2 — DEPLOY TO RAILWAY (Recommended — Free tier)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   Files needed in your GitHub repo:
   ├── alchemist_server.py
   ├── requirements.txt
   └── Procfile

   1. Create a FREE GitHub account at github.com
   2. Create a new repository (name it "alchemist-server")
   3. Upload the 3 files above

   4. Go to: https://railway.app
   5. Sign up with GitHub
   6. Click "New Project" → "Deploy from GitHub repo"
   7. Select your "alchemist-server" repo
   8. Railway auto-detects Python and Procfile ✅

   9. Add environment variables (click "Variables" tab):
      ANTHROPIC_API_KEY  = sk-ant-...your key...
      TELEGRAM_BOT_TOKEN = 123456:ABC...your token...
      TELEGRAM_CHAT_ID   = your_chat_id

   10. Click "Deploy" — takes ~60 seconds
   11. Go to "Settings" → "Networking" → "Generate Domain"
       Your URL will be: https://alchemist-server-xxxx.railway.app

   🧪 TEST IT:
      Open: https://your-url.railway.app/
      Should return: {"status": "online", "service": "The Alchemist..."}

      POST test signal:
      curl -X POST https://your-url.railway.app/test
      → Fires EURUSD BUY through Claude → Telegram

STEP 3 — CONNECT TRADINGVIEW ALERTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   (Requires TradingView Pro/Pro+/Premium for webhooks)

   1. Add The Alchemist Pine Script to your chart
      (TradingView → Pine Editor → paste TheAlchemist_MSNR_SMC_ICT.pine → Add to chart)

   2. Click the Clock icon (Alerts) on TradingView

   3. Create Alert for BUY signals:
      Condition:      "Alchemist BUY Setup"
      Message:        (leave empty — Pine Script sends its own JSON)
      Notifications:  ✅ Webhook URL
      Webhook URL:    https://your-url.railway.app/webhook

   4. Create Alert for SELL signals:
      Condition:      "Alchemist SELL Setup"
      Webhook URL:    https://your-url.railway.app/webhook

   5. Create Alert for BOS confirmation:
      Condition:      "Alchemist — Bullish BOS (Confirmation)"
      Webhook URL:    https://your-url.railway.app/webhook/bos

   6. Repeat step 5 for Bearish BOS

   ✅ Done! Every signal now flows:
      TradingView → Your Server → Claude AI → Telegram

   3. Build Command:  pip install -r requirements.txt
   4. Start Command:  gunicorn alchemist_server:app --bind 0.0.0.0:$PORT
   5. Add environment variables (same as Railway)
   6. Deploy → get your .onrender.com URL

   NOTE: Free tier on Render spins down after 15 min inactivity.
   Use Railway for always-on reliability.

TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ❌ "Claude Analysis Unavailable"
     → Check ANTHROPIC_API_KEY in Railway variables
     → Make sure key starts with sk-ant-

  ❌ Telegram not receiving messages
     → Make sure you sent a message to your bot first
     → Check TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID
     → For groups: bot must be admin in the group

  ❌ TradingView says "Webhook Error"
     → URL must be HTTPS (Railway gives this automatically)
     → Test URL manually: curl https://your-url/

  ❌ No signals firing
     → Make sure you're in London (07-10 GMT) or NY (12-14 GMT) session
     → The Pine Script only fires during killzones by design
     → During testing: temporarily remove "in_killzone" condition

  ✅ Test at any time by POSTing to /test endpoint
